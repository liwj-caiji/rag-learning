## 7. RAGAS 评估模块

### 7.1 概述

`src/evaluation/` 模块封装 RAGAS 评估流水线，在每次回答生成后自动计算 5 项标准指标，量化检索和生成质量。

### 7.2 模块结构

```
src/evaluation/
├── __init__.py       # 公开 API（load_eval_dataset, RAGASEvaluator, 报告函数）
├── config.py         # 评估专用配置（LLM、指标列表、批处理参数）
├── evaluator.py      # RAGASEvaluator 核心评估器
├── dataset.py        # YAML 数据集加载、意图过滤
└── reporter.py       # 控制台 / JSON / HTML 报告生成
```

### 7.3 评估流水线

```
evaluator.evaluate(samples, metrics)
  │
  ├── 1. 对每个样本调用 pipeline.trace(query)
  │     ├── query → rewrite → retrieve → generate
  │     └── 提取 user_input, response, retrieved_contexts, reference
  │
  ├── 2. 构建 RAGAS Dataset（datasets.Dataset）
  │
  ├── 3. 调用 ragas.evaluate(dataset, metrics, llm, embeddings)
  │     ├── ContextPrecision  ─── 上下文精确度
  │     ├── ContextRecall     ─── 上下文召回率
  │     ├── Faithfulness      ─── 忠实度
  │     ├── AnswerRelevancy   ─── 回答相关性
  │     └── AnswerCorrectness ─── 回答正确性（可选）
  │
  └── 4. 聚合结果
        ├── 整体分数（各指标均值）
        ├── 按意图分组（推荐 / 做法 / 原料 / 事实）
        └── 逐样本评分
```

### 7.4 评估指标说明

| 指标 | 含义 | 依赖 | 当前值 |
|------|------|------|:---:|
| **answer_relevancy** | 回答与原始问题的相关程度 | LLM + Embeddings | 0.712 |
| **context_precision** | 检索到的上下文中，实际有用的比例 | LLM + ground_truth | 0.442 |
| **context_recall** | ground_truth 中有多少能在上下文中找到 | LLM + ground_truth | 0.765 |
| **faithfulness** | 回答中的陈述能否从上下文推断（反幻觉） | LLM | 0.817 |
| **answer_correctness** | 回答与 ground_truth 的事实一致性 | LLM + Embeddings + ground_truth | — |

> ground_truth 已在 `data/evaluation/test_queries.yaml` 中与 `base/HowToCook` 实际食谱文件对齐。

### 7.5 按意图分组评估

| 意图 | 样本数 | answer_rel | context_prec | context_rec | faithfulness |
|------|:---:|:---:|:---:|:---:|:---:|
| 推荐 | 10 | 0.844 | 0.100 | — | 0.783 |
| 做法查询 | 10 | 0.736 | 0.700 | 0.875 | 0.973 |
| 原料查询 | 6 | 0.796 | 0.750 | 1.000 | 0.710 |
| 事实查询 | 4 | 0.000 | 0.000 | 0.000 | 0.659 |

**解读**：
- **做法查询** 各项指标最优（faithfulness 0.973），说明检索+生成的菜谱做法质量高、忠实度好
- **推荐** context_precision 低（0.100），属于结构性原因：推荐策略检索大量候选（~15），LLM 只选取 4-5 个推荐，大部分检索结果对最终回答"无用"
- **事实查询** 三项为 0，原因是知识库为菜谱集合，不含"老抽和生抽的区别""豆瓣酱是什么"等百科知识

## 8. RAGAS 中文提示词适配

### 8.1 问题

RAGAS 默认评估提示词为英文。用英文提示词评估中文问答系统时，LLM 在理解中文上下文、判断中文语义相关性时表现不佳，导致 context_precision、context_recall 等指标分数异常偏低。

### 8.2 方案

采用 ragas 官方 `adapt_prompts(language="chinese")` API，使用评估 LLM（deepseek-chat）将每个指标的提示词自动翻译为中文。

```python
async def _adapt():
    for metric in metrics_list:
        adapted = await metric.adapt_prompts(language="chinese", llm=self.eval_llm)
        metric.set_prompts(**adapted)

asyncio.run(_adapt())
```

### 8.3 缓存机制

首次运行后，翻译结果通过 pickle 持久化到 `.ragas_cache/chinese_prompts.pkl`，后续运行直接加载，零额外 API 调用：

```
_adapt_prompts_chinese(metrics_list)
  ├── 缓存文件存在？ → 加载并 set_prompts → 完成
  └── 缓存不存在？
        ├── 调用 adapt_prompts(language="chinese") × N 个指标
        ├── pickle.dump → .ragas_cache/
        └── 完成
```

### 8.4 方案对比

| 方案 | 代码量 | 维护成本 | 翻译质量 |
|------|:---:|:---:|:---:|
| 环境变量 `RAGAS_LANGUAGE=chinese` | 0 行 | 无 | 依赖版本支持 |
| 手写 PydanticPrompt（已废弃） | 398 行 | 高（需与 ragas 内部 schema 对齐） | 可控 |
| **adapt_prompts + 缓存（当前方案）** | **40 行** | **低（官方 API）** | **LLM 翻译** |

## 9. 评估数据集

### 9.1 数据集设计

`data/evaluation/test_queries.yaml` 包含 30 条中文烹饪领域查询，覆盖 4 种意图类型：

| 意图 | 示例 | 数量 | 含 ground_truth |
|------|------|:---:|:---:|
| howto（做法查询） | "麻婆豆腐怎么做"、"清蒸鲈鱼怎么做好吃" | 10 | ✓ |
| ingredient（原料查询） | "可乐鸡翅需要什么材料"、"红烧肉需要哪些调料" | 6 | ✓ |
| recommendation（推荐） | "今天吃什么"、"适合新手的菜有哪些" | 8 | — |
| factual（事实查询） | "老抽和生抽有什么区别"、"豆瓣酱是什么" | 6 | 部分 |

每条数据字段：

```yaml
- query: "麻婆豆腐怎么做"
  intent: howto
  target_dish: "麻婆豆腐"
  ground_truth: |
    麻婆豆腐的主要原料：内脂豆腐、咸鸭蛋、五花肉...
    步骤：1.大蒜生姜切碎...
```

### 9.2 ground_truth 对齐

v1.2 的 ground_truth 已与 `base/HowToCook` 中**实际食谱文件**逐条对齐：

- 麻婆豆腐使用「咸鸭蛋 + 香辣酱」版本（非传统豆瓣酱版）
- 水煮鱼使用「巴沙鱼」版本（非传统草鱼版）
- 宫保鸡丁使用「手枪腿」版本（含腌制 1 小时流程）
- 番茄炒蛋、饺子等知识库不存在的菜品不设 ground_truth，评估时 context_recall 自动跳过（nan）

## 10. 评估脚本

### 10.1 evaluate.py

```bash
# 完整评估（LLM 模式）
uv run python scripts/evaluate.py --mode llm

# 仅评估做法查询
uv run python scripts/evaluate.py --mode llm --intent howto

# 限制样本数（快速验证）
uv run python scripts/evaluate.py --mode llm --limit 5

# 导出 JSON 报告
uv run python scripts/evaluate.py --mode llm --output report.json
```

### 10.2 generate_report.py

```bash
uv run python scripts/generate_report.py report.json -o docs/evaluation.html
```

### 10.3 Python API

```python
from src.evaluation import load_eval_dataset, RAGASEvaluator
from src.generation import RAGPipeline

pipeline = RAGPipeline(use_llm=True)
evaluator = RAGASEvaluator(pipeline)

samples = load_eval_dataset("data/evaluation/test_queries.yaml")
result = evaluator.evaluate(samples, metrics=["context_precision", "context_recall", "faithfulness"])

print(result.aggregate)  # {"context_precision": 0.44, ...}
for sr in result.samples[:3]:
    print(sr.query, sr.scores)
```

## 附录 F（v1.2 新增）：关键设计决策（v1.2 追加）

1. **adapt_prompts 替代手写提示词**：ragas 0.4.3 官方 API 自动翻译，比手写 PydanticPrompt（398 行）更易维护，翻译质量经 LLM 验证与手写版相当
2. **ground_truth 与知识库对齐**：评估数据集的 ground_truth 必须与实际检索到的食谱文件一致，否则 context_recall 会系统性偏低（修正前 0.250 → 修正后 0.765）
3. **磁盘缓存避免重复 API 调用**：`adapt_prompts` 翻译结果 pickle 到 `.ragas_cache/`，首次运行后零额外成本
4. **推荐类 context_precision 结构性偏低**：推荐策略检索大量候选再精选，大部分检索结果天然与最终答案"无关"，此指标对推荐场景参考价值有限
